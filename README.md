## カテゴリの分類方針

### 03-設計 と 04-開発手法 の違い

このリポジトリでは `03-設計/` と `04-開発手法/` を意図的に分離しています。
一見似ているように見えますが、扱う関心事が異なります。

| 観点 | 03-設計 | 04-開発手法 |
| --- | --- | --- |
| **問い** | 何をどう**構造化**するか？ | どう**進める**か？ |
| **関心事** | システムの構造・見た目・振る舞いの定義 | 開発プロセス・進め方・ワークフロー |
| **具体例** | アーキテクチャ、デザインパターン、UML、UI/UX | アジャイル、仕様書駆動開発、API開発プロセス、コードリーディング |

#### 具体例で理解する

- 「MVCパターンでコントローラとモデルを分離する」→ **03-設計**（構造の決定）
- 「仕様書を先に書いてからコードを実装する」→ **04-開発手法**（進め方の決定）
- 「ERDでテーブル関係を定義する」→ **03-設計**（データ構造の設計）
- 「スプリント単位でタスクを回す」→ **04-開発手法**（開発の進め方）

#### 迷ったときの判断基準

> **「成果物の形を決める話」なら `03-設計/`、「作り方の手順を決める話」なら `04-開発手法/`** に分類してください。

### 04-開発手法 と 07-実務経験・振り返り の違い

`04-開発手法/` は、スクラム、TDD、仕様書駆動開発のような再利用可能な方法論やプロセスを扱います。
`07-実務経験・振り返り/` は、プロジェクト参画や日々の業務から得た個別具体的な学び、課題、改善点を扱います。

| 観点 | 04-開発手法 | 07-実務経験・振り返り |
| --- | --- | --- |
| **問い** | どう進めるか？ | 実務から何を学び、次にどう活かすか？ |
| **関心事** | 開発プロセス・進め方・ワークフロー | 経験、反省、改善、再発防止 |
| **具体例** | スクラム、TDD、仕様書駆動開発 | 日次の課題記録、プロジェクト参画の振り返り |

各フォルダ内に詳細なドキュメントやサンプルコード、設計資料などを格納しています。
