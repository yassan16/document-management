## Phase 1: デザイン基礎

### 1.1 視覚デザインの基本原則

#### CRAP原則

- **Contrast（コントラスト）**: 要素間の違いを明確にする
  - サイズ、色、太さで差をつける
  - 重要な情報を目立たせる
- **Repetition（反復）**: 一貫したデザイン要素を繰り返す
  - ブランドの統一感を生む
  - ユーザーの学習コストを下げる
- **Alignment（整列）**: 要素を意図的に配置する
  - 見えない線で要素を結ぶ
  - 視線の流れを作る
- **Proximity（近接）**: 関連する要素をグループ化する
  - 情報の関連性を視覚的に示す
  - 余白で区切りを表現

#### 視覚的階層（Visual Hierarchy）

- サイズ: 大きい要素ほど重要に見える
- 色・コントラスト: 目立つ色で注目を集める
- 位置: 左上から右下への視線の流れ（Z/Fパターン）
- タイポグラフィ: 見出し・本文・キャプションの区別

#### グリッドシステム

- 8ptグリッドの基本
- 12カラムグリッドレイアウト
- レスポンシブグリッドの考え方
- 余白（margin/padding）の一貫性

#### 学習目標

- [ ] CRAP原則を使ってデザインを分析できる
- [ ] 視覚的階層を意図的に設計できる
- [ ] グリッドシステムに沿ったレイアウトを作成できる

### 1.2 タイポグラフィ

#### フォントの基礎

- セリフ体 vs サンセリフ体の使い分け
- ウェイト（太さ）のバリエーション
- フォントファミリーの選び方
- Webフォントとシステムフォント

#### 可読性の確保

- 行間（line-height）: 1.4〜1.6が目安
- 行長（line-length）: 50〜75文字程度
- 文字サイズ: 本文は16px以上
- コントラスト比: WCAG基準（4.5:1以上）

#### タイポグラフィスケール

```
見出し1: 32px (2rem)
見出し2: 24px (1.5rem)
見出し3: 20px (1.25rem)
本文:    16px (1rem)
キャプション: 14px (0.875rem)
```

#### 学習目標

- [ ] 適切なフォントを選択できる
- [ ] 可読性の高いテキストデザインができる
- [ ] 一貫したタイポグラフィスケールを設計できる

### 1.3 カラー理論

#### 色の基礎

- 色相（Hue）、彩度（Saturation）、明度（Lightness）
- 暖色と寒色の心理効果
- 色のアクセシビリティ

#### 配色パターン

- モノクロマティック（単色）
- アナロガス（類似色）
- コンプリメンタリー（補色）
- トライアド（三色配色）

#### UIにおける色の使い方

- プライマリカラー: ブランドの主要色
- セカンダリカラー: アクセントやサポート
- ニュートラルカラー: 背景・テキスト
- セマンティックカラー: 成功（緑）、警告（黄）、エラー（赤）、情報（青）

#### ダークモード対応

- 純粋な黒（#000）は避ける
- コントラストの調整
- セマンティックカラーの明度調整

#### 学習目標

- [ ] 調和のとれた配色を作成できる
- [ ] アクセシビリティに配慮した色選びができる
- [ ] ライト/ダークモード両対応の配色を設計できる

### 1.4 UXの基本概念

#### ユーザー中心設計（UCD）

- ユーザーのニーズを起点に設計する
- 仮説→検証→改善のサイクル
- デザインはアートではなく問題解決

#### ニールセンのユーザビリティ10原則

1. システム状態の可視性
2. システムと現実世界の一致
3. ユーザーのコントロールと自由
4. 一貫性と標準
5. エラーの防止
6. 記憶より認識
7. 柔軟性と効率性
8. 美的で最小限のデザイン
9. エラーからの回復支援
10. ヘルプとドキュメント

#### アクセシビリティ（WCAG）

- 知覚可能（Perceivable）
- 操作可能（Operable）
- 理解可能（Understandable）
- 堅牢（Robust）

#### 学習目標

- [ ] ユーザー中心設計の考え方を説明できる
- [ ] ユーザビリティ10原則でUIを評価できる
- [ ] 基本的なアクセシビリティ要件を満たせる

## Phase 3: UXリサーチ手法

### 3.1 ユーザー理解

#### ペルソナ作成

- 定量データ（アンケート、アナリティクス）の収集
- 定性データ（インタビュー、観察）の収集
- 行動パターンの分類
- ペルソナドキュメントの作成

```
【ペルソナ例】
名前: 田中 健太（32歳）
職業: マーケティング担当者
目標: 業務効率化で残業を減らしたい
課題: ツールが多すぎて情報が分散している
技術レベル: 中級（基本的なツールは使いこなす）
```

#### ユーザーインタビュー

- インタビューガイドの準備
- オープンクエスチョンの活用
- 5 Whys（なぜを5回繰り返す）
- バイアスを避ける質問技法

#### ジャーニーマップ

- ユーザーの行動フローを可視化
- タッチポイントの特定
- 感情の変化を記録
- ペインポイントと機会の発見

#### 学習目標

- [ ] データに基づいたペルソナを作成できる
- [ ] 効果的なユーザーインタビューを実施できる
- [ ] ジャーニーマップで課題を可視化できる

## Phase 5: 実践とポートフォリオ

### 5.1 模写からオリジナルへ

#### Step 1: UI模写

- 人気アプリのスクリーンショットを再現
- ピクセルパーフェクトを目指す
- デザインの意図を考えながら作業

#### Step 2: リデザイン

- 既存アプリの改善案を作成
- Before/Afterで比較
- 改善理由を言語化

#### Step 3: オリジナル作成

- 自分で課題を設定
- リサーチ→設計→検証のプロセスを実践
- デザイン判断を記録

#### 学習目標

- [ ] 既存のUIを正確に模写できる
- [ ] 根拠のある改善提案ができる
- [ ] ゼロからプロジェクトを完遂できる

-----

### 5.2 ポートフォリオ構築

#### プロジェクト構成

1. **概要**: 何を作ったか、役割は何か
2. **課題**: 解決しようとした問題
3. **プロセス**: リサーチ、設計、検証の流れ
4. **成果物**: 最終デザイン
5. **振り返り**: 学びと改善点

#### ケーススタディの書き方

- ストーリーとして読めるように
- プロセスを見せる（最終成果だけでなく）
- データや根拠を示す
- Before/Afterを効果的に使う

#### ポートフォリオサイト

- 作品の選定（3〜5プロジェクト）
- 自己紹介
- 連絡先
- 読みやすいレイアウト

#### 学習目標

- [ ] プロセスを含むケーススタディを書ける
- [ ] 効果的なポートフォリオを構築できる
- [ ] 自分のデザイン判断を説明できる

## 学習のコツ

### 1. 毎日手を動かす

知識のインプットだけでなく、毎日何かをデザインする習慣をつける。Daily UIなどのチャレンジを活用。

### 2. 良いデザインを分析する

普段使うアプリやWebサイトを「なぜこうなっているのか」という視点で観察する。スクリーンショットを集めてライブラリ化。

### 3. フィードバックを得る

- デザインコミュニティへの投稿
- メンターからのレビュー
- ユーザーテストの実施

### 4. 実装の基礎を理解する

HTML/CSSの基礎があると、開発者との連携がスムーズになり、実現可能なデザインを提案できる。

### 5. プロセスを大切にする

最終成果物だけでなく、リサーチ→設計→検証のプロセスを丁寧に行う。ポートフォリオでもプロセスを見せる。

### 6. 言語化する習慣

デザイン判断の理由を常に言葉にする。「なんとなく」ではなく、根拠を持った意思決定を。
