# 課題整理メモ

## 概要

日々の業務で気になったことや、後で月次課題記録に整理したい内容を一時的に書き留めるためのメモです。
まだ整理できていない内容でも、忘れないうちに残しておくことを目的とします。

### 2026-07-09

- Docsリポジトリの作成が大変で、構築が進みづらい。
  - 現在は docs -> FE -> BE の順で進めているが、docs に記載する量が多く、CIチェックで指摘される内容も多いため、開発メンバーの負担になっている。
  - 理想的なドキュメントと、実際に開発を進められる運用で必要なドキュメントには差があるのではないか。
  - ドキュメントの品質を保つことと、開発が滞らない粒度に抑えることのバランスを整理したい。
