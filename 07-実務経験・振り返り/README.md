## 概要

実務経験から得た学び、課題、改善点を記録するためのドキュメントを整理します。
プロジェクト参画や日々の業務を通じて得た気づきを、次の行動や仕組みに落とし込むことを目的とします。
次回同じ問題に遭遇したときに、同じ間違いを繰り返さないための判断基準や対策を残します。

## 運用方針

- 事実、原因、次回の判断基準、仕組み化を分けて記録する
- 人ではなく、状況・判断・プロセス・情報の流れに着目する
- 反省だけで終わらせず、次回の行動やチェックリストに落とし込む
- 継続的に見返せるように、日付ごとの記録形式をそろえる

## 日々の運用方法

日々の記録は、月ごとにMarkdownファイルを作成して管理します。
`月次課題記録テンプレート.md` は書き方の基準として残し、実際の記録ファイルには日付ごとの記録ブロックを追記します。

ファイル名は年月が分かる形式にします。

```text
07-実務経験・振り返り/
├── README.md
├── 月次課題記録テンプレート.md
├── 2026-07.md
├── 2026-08.md
└── 2026-09.md
```

運用の流れは次の通りです。

1. 月初に `YYYY-MM.md` 形式のファイルを作成する
2. その日の業務で最も改善効果が高い課題を1つ選ぶ
3. テンプレートの記録ブロックをコピーし、日付見出しとして追記する
4. `原因` で止めず、`次回の判断基準` と `仕組み化` まで書く
5. 月末に記録を見返し、繰り返し発生している課題や改善できた点を整理する

月次ファイルの末尾には、必要に応じて次のような振り返り欄を追加します。

```markdown
## 月次振り返り

### 繰り返し発生した課題

-

### 改善できたこと

-

### 来月の重点改善テーマ

-
```
