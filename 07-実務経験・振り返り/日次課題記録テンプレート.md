# 日次課題記録テンプレート

## 概要

プロジェクト参画中に発生した課題を1日1点記録し、原因・再発防止・仕組み化の観点で整理するためのテンプレートです。
その日に起きた問題を単なる反省で終わらせず、次回の判断基準や作業プロセスの改善に変換することを目的とします。

## 記録ルール

- 1日につき、最も改善効果が高い課題を1つだけ記録する
- 起きた事実と解釈を分けて書く
- 個人の注意不足で終わらせず、判断基準や仕組みに落とす
- 人ではなく、状況・判断・プロセス・情報の流れに着目する
- 次のアクションは、実行できる大きさに分解する

## 記録テンプレート

```markdown
## YYYY-MM-DD

### 今日の課題

-

### 起きたこと

-

### 原因

-

### 次回の判断基準

-

### 仕組み化

-

### 次のアクション

- [ ]
```

## 記入例

```markdown
## 2026-07-09

### 今日の課題

- 仕様の前提を確認しないまま実装に入り、手戻りが発生した

### 起きたこと

- 既存仕様の確認が不十分な状態で修正方針を決めた
- レビュー時に想定していなかった利用ケースを指摘された
- 実装の一部をやり直す必要が出た

### 原因

- 作業前に確認すべき仕様・画面・関連コードを洗い出していなかった
- 不明点を質問する前に、自分の前提を明文化していなかった

### 次回の判断基準

- 仕様が不明なまま実装に入らない
- 影響範囲が2ファイル以上に広がる場合は、先に修正方針を確認する
- 15分調べても判断できない場合は、調査内容と仮説をまとめて相談する

### 仕組み化

- 作業開始前に「目的 / 影響範囲 / 不明点」をメモする
- 相談時は「事象 / 試したこと / 仮説 / 確認したいこと」の形式で書く
- PR作成前チェックリストに「仕様の前提を書いたか」を追加する

### 次のアクション

- [ ] 作業前メモのテンプレートを作る
- [ ] PR作成前チェックリストに仕様確認の項目を追加する
```
