# CLAUDE.md

このドキュメント集の作成・編集ルールです。

---

## WHAT（これは何か）

- エンジニアとしてのナレッジを蓄積していくためのドキュメント群

---

## WHY（なぜ必要か）

- エンジニアとして学習成長するため

---

## HOW（どう書くか）

### ドキュメント作成

| 操作              | 方法                                                         |
| ----------------- | ------------------------------------------------------------ |
| 新規ドメイン追加  | `{domain}/*/{任意のファイル名}.md` を作成し、README.mdに追加 |
| README.mdへの反映 | ファイルの作成・変更時は関連する`README.md` を更新する       |

### 図表の使い分け

| 図表              | 用途                       |
| ----------------- | -------------------------- |
| `erDiagram`       | テーブル関係・カラム定義   |
| `sequenceDiagram` | API呼び出し・処理フロー    |
| `flowchart`       | 判定ロジック・状態遷移     |
| Markdownテーブル  | カラム説明・ステータス一覧 |

### ドキュメント構造

- 概要、目次、以降に詳細説明を追記すること
- #, ##, ### 見出しを適切に使い分け、必ず階層構造を守り、目次のリンクを貼ること

### 詳細の書き方

- 説明の後に、なぜ？の観点を入れること

---

## SuperClaudeの使用

アウトプットの精度を向上させるため、**SuperClaude**スキルを必ず使用してください。以下に例を記載します。

| スキル           | 用途                             |
| ---------------- | -------------------------------- |
| `/sc:document`   | ドキュメント生成                 |
| `/sc:analyze`    | コード分析・品質チェック         |
| `/sc:explain`    | コード・概念の説明               |
| `/sc:index-repo` | リポジトリ全体のインデックス作成 |

---

## 参照ファイル

- **README.md** - 全体構成・技術スタック
- **business-rules.md** - ビジネスルール・主要概念
- **{domain}/index.md** - 各ドメインの詳細
